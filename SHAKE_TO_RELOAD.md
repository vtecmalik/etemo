# Shake to Reload - Встряхивание для перезагрузки

## Что это?

Функция "Shake to Reload" позволяет перезагружать приложение простым встряхиванием устройства. Это очень удобно во время разработки и тестирования!

## Как использовать?

### В режиме разработки (Development)

1. Откройте приложение на физическом устройстве
2. Встряхните устройство
3. Появится alert "Перезагрузка приложения..."
4. Приложение автоматически перезагрузится через 0.5 секунды

**Примечание:** В симуляторе встряхивание не работает. Используйте физическое устройство.

### В режиме продакшн (Production build)

1. Встряхните устройство
2. Если есть обновление - появится диалог с предложением обновить
3. Если обновлений нет - приложение просто перезагрузится

## Технические детали

- **Порог чувствительности:** 2.5G (ускорение)
- **Интервал проверки:** 100мс
- **Используется:** expo-sensors (Accelerometer)

## Настройка чувствительности

Если хотите изменить чувствительность встряхивания, отредактируйте в `App.tsx`:

```typescript
const SHAKE_THRESHOLD = 2.5; // Увеличьте для меньшей чувствительности
```

## Что еще добавлено?

### ErrorBoundary
Приложение теперь ловит все необработанные ошибки и показывает понятное сообщение вместо белого экрана.

### Loading Screen
Добавлен экран загрузки при инициализации приложения.

### Navigation Error Handling
Обработка ошибок навигации для предотвращения крашей.

## Исправлена проблема с белым экраном

Теперь если при запуске приложения что-то пойдет не так:

1. **Не будет белого экрана** - вы увидите сообщение об ошибке
2. **Можно встряхнуть устройство** для перезагрузки
3. **Ошибки логируются** в консоль для отладки

## Тестирование

Чтобы протестировать shake to reload:

```bash
# Запустите на физическом устройстве
npm start

# Отсканируйте QR-код в Expo Go
# Или соберите development build:
npx expo run:android --variant debug
npx expo run:ios --configuration Debug
```

Затем просто встряхните устройство!

## Troubleshooting

**Встряхивание не работает:**
- Убедитесь, что используете физическое устройство (не симулятор)
- Встряхните сильнее
- Попробуйте уменьшить SHAKE_THRESHOLD до 2.0

**Приложение перезагружается слишком часто:**
- Увеличьте SHAKE_THRESHOLD до 3.0 или выше
- Увеличьте UPDATE_INTERVAL

**Приложение не показывает обновления в продакшн:**
- Проверьте настройки expo-updates в app.json
- Убедитесь, что опубликовали обновление через `eas update`
